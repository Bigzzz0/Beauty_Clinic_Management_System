# Palette's Journal

## 2026-02-17 - Focus Visible
**Learning:** Custom interactive elements (like cards acting as links) often miss default browser focus styles.
**Action:** Always add `focus-visible` utility classes to non-button interactive elements.

## 2026-02-17 - Dashboard Empty States
**Learning:** Positive empty states (e.g., "All Stock Healthy") reinforce good system status instead of just looking broken or empty.
**Action:** Always include a positive confirmation icon/text for "Zero Errors" or "Zero Warnings" states.

## 2026-02-17 - Input Autocomplete
**Learning:** Adding `autoComplete` attributes is a zero-cost high-value improvement for accessibility and usability (password managers).
**Action:** Always verify `autoComplete` is present on authentication forms.

## 2026-02-17 - Button Loading State
**Learning:** Standardizing loading states in the Button component reduces boilerplate code in forms and ensures visual consistency.
**Action:** Use `isLoading` prop for all future async buttons instead of manual state handling.
